# Правила проекта DocuSpec

## Язык
- Общение только на русском языке.

## Планирование
- План должен быть высокоуровневым и архитектурным.
- Указывать необходимость изменения БД, но без технических деталей реализации.

## База данных (Supabase)
- RLS не использовать.
- Миграции выполняются через SQL Editor в облачном Supabase.
- При доработке НЕ дописывать в существующий файл миграции.
- ВСЕГДА создавать новую миграцию (sql/002_..., sql/003_... и т.д.) с расчётом на то, что предыдущая уже применена в Supabase.
- Новая миграция должна учитывать текущее состояние БД после всех предыдущих миграций.

## Файлы
- MD файлы не сохранять в корне проекта.
- Обычные отчёты — в `temp/`.
- Архитектурная документация — в `docs/`.

## Git
- Самостоятельно не создавать коммиты.
- Не пушить в GitHub.
